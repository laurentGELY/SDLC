Raison d'être

Ce que le modèle résout

Sans gouvernance explicite, Claude Code opère sur des règles implicites — qui disparaissent entre les sessions. Ce modèle rend les contraintes non-négociables, les décisions traçables, et les procédures reproductibles.

01
Vérification exécutable
Tout plan de test contient une commande exacte. Claude itère seul jusqu'à ce que la commande passe — il ne déclare jamais "terminé" sur estimation.
Signal d'alarme : si Claude peut conclure sans preuve observable → violation.
02
Circuit fermé
Toute règle implicite devient explicite (hook, DECISIONS.md, ou Claude.md). Si une contrainte dépend de la mémoire du modèle entre sessions, elle n'existe pas opérationnellement.
Signal d'alarme : observation dans LESSONS_LEARNED sans "Action proposée" → circuit ouvert.
03
Contexte chirurgical
Chaque template liste explicitement les fichiers à charger. Claude ne lit pas le repo en entier — il charge uniquement ce qui est listé dans §Handoff.
Signal d'alarme : template qui demande cat specs/SPEC.md sans condition → lecture non-chirurgicale.
04
Boucle de rétroaction
Toute observation de terrain a un chemin vers une règle permanente. L'ordre : incident → LESSONS_LEARNED → /retrospective → décision → hook ou Claude.md.
Signal d'alarme : règle dans pre-tool-bash.sh sans entrée DECISIONS.md → règle implicite.

Mécanismes

Les trois circuits fermés

Chaque circuit transforme une observation de terrain en règle permanente. Aucun ne dépend de la mémoire du modèle entre sessions.

Sprint 0

Bootstrapper un nouveau projet

Le Sprint 0 installe la gouvernance complète avant le premier sprint métier. 9 étapes, un seul commit final.

Initialiser le repo
Créer le dépôt git, commit vide initial.
git init && git commit --allow-empty -m "chore: init repo"
Créer la structure des dossiers
doc/, specs/Sprints/, .claude/skills/wrap-up/, .claude/skills/retrospective/, .claude/hooks/, scripts/
Copier sans modification
specs/sprint-template.md (depuis 04) · .claude/skills/retrospective/SKILL.md (depuis 09)
Adapter Claude.md et STANDARDS.md
Remplir §Rôle, §Démarrage §2, §Limites bash, §Fichier .env. Vider §Carte des étapes et §Modules partagés.
grep -i "\[→ adapter\]" Claude.md → zéro résultat attendu
Adapter wrap-up SKILL
§Étape 0b (chemins git), §Étape 6 (nom projet Claude.ai), déclencheurs §Étape 3 conditionnels.
Créer les fichiers from scratch
CHANGELOG.md · doc/DECISIONS.md · doc/ROADMAP.md · doc/LESSONS_LEARNED.md (avec format Hook + SDLC candidat) · doc/DIAGNOSTIC_CMDS.md · specs/SPEC.md · .claude/skills/diagnostic/SKILL.md
Configurer les hooks
Copier pre-tool-bash.sh, décider des sections [ACTIVER si…], documenter chaque choix dans DECISIONS.md §D-HOOK-XX, test smoke.
grep "\[ACTIVER" .claude/hooks/pre-tool-bash.sh → zéro résultat
Vérification finale
11 critères d'acceptation. Zéro placeholder résiduel. Test smoke hook exit 0.
Commit
Un seul commit final après validation des 11 critères.
git commit -m "chore: bootstrap SDLC v1.2"
